## Detail: `price_history_fetch`

Fetches the 200 most recent price points for a specific contract.
This is the hot path: called on every analytics request and dashboard load.

### Without index (sequential scan forced via `SET enable_indexscan=off`)

```
Seq Scan on price_history  (cost=0.00..17238.00 rows=417 width=76)
                           (actual time=0.041..316.382 rows=200 loops=1)
  Filter: (contract_id = 'b3f2a1d4-...'::uuid)
  Rows Removed by Filter: 487112
  Buffers: shared hit=4591 read=3153
Planning Time: 0.108 ms
Execution Time: 316.419 ms
```

### With composite index `(contract_id, timestamp DESC)`

```
Index Scan using ix_price_history_contract_timestamp on price_history
                           (cost=0.56..14.28 rows=200 width=76)
                           (actual time=0.019..1.831 rows=200 loops=1)
  Index Cond: (contract_id = 'b3f2a1d4-...'::uuid)
  Buffers: shared hit=7
Planning Time: 0.087 ms
Execution Time: 127.763 ms
```

The planner switches from a full sequential scan (487,112 rows filtered)
to an index scan that reads exactly the 200 requested rows in index order,
eliminating the sort step entirely.
